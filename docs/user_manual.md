# 物联网多源传感器数据分析系统 — 用户手册

## 1. 系统概述

物联网多源传感器数据分析系统（以下简称"本系统"）是一款面向物联网领域的桌面数据分析软件，支持多来源传感器数据的接入、清洗、分析、可视化与报告导出，帮助用户快速掌握传感器数据的变化规律和异常情况。

### 1.1 主要功能

- **数据接入**：支持 CSV 文件导入、串口实时读取、MQTT 物联网协议订阅
- **数据清洗**：缺失值填充、异常值检测与过滤、数据平滑
- **统计分析**：描述性统计、相关性分析、趋势分析、分布检验
- **可视化**：时间序列图、散点图、热力图、直方图、箱线图、综合仪表盘
- **报告导出**：Excel 报告（含图表）、纯文本报告

### 1.2 运行环境要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10 / Linux / macOS | Windows 11 |

| 内存 | 4 GB | 8 GB |
| 磁盘空间 | 200 MB | 500 MB |
| 显示器分辨率 | 1280 × 720 | 1920 × 1080 |

## 2. 安装与启动

### 2.1 安装环境

确保已安装 Python 3.9 或更高版本：

```bash
python --version
```

### 2.2 安装依赖

进入软件目录，使用 pip 安装所有依赖：

```bash
cd iot-sensor-analysis
pip install -r requirements.txt
```

### 2.3 启动软件

**图形界面模式（推荐）：**

```bash
python main.py
```

**命令行批处理模式：**

```bash
python main.py --cli samples/sample_sensor_data.csv
```

## 3. 图形界面操作指南

### 3.1 主窗口布局

启动后显示主窗口，包含以下部分：

- **菜单栏**：文件、处理、分析、视图、帮助
- **工具栏**：常用功能快捷按钮
- **标签页**：数据预览、统计分析、可视化、处理日志
- **状态栏**：显示当前操作状态

[图 3-1 主窗口界面]

### 3.2 导入数据

**从 CSV 文件导入：**

1. 点击菜单"文件 → 导入 CSV 文件"或工具栏"导入CSV"按钮
2. 在文件对话框中选择 CSV 文件
3. 系统自动识别编码格式、分隔符和时间戳列
4. 数据导入后在"数据预览"标签页显示

**加载示例数据：**

点击"文件 → 导入示例数据"可快速加载内置的传感器示例数据，包含温度、湿度、气压、光照、风速五个传感器，共 2000 条记录，部分数据含手动注入的异常值。

[图 3-2 数据导入界面]

### 3.3 数据处理

#### 3.3.1 缺失值填充

点击"处理 → 缺失值填充"或工具栏对应按钮：

- 选择"是"用线性插值填充
- 选择"否"用前向填充
- 系统支持 6 种填充方法：前向填充、后向填充、线性插值、均值填充、中位数填充、零值填充

#### 3.3.2 异常值检测

点击"处理 → 异常值检测"：

系统使用 3σ（拉依达准则）检测异常值，即在均值 ±3 倍标准差范围之外的数值标记为异常。检测结果在"统计分析"标签页展示，包括：

- 各传感器异常值数量及占比
- 正常数值范围（上界/下界）
- 均值与标准差

[图 3-3 异常检测结果]

#### 3.3.3 数据平滑

点击"处理 → 数据平滑"使用移动平均滤波，默认窗口大小为 5。也可通过代码调用 Savitzky-Golay 滤波器进行更高级的平滑处理。

#### 3.3.4 运行处理管线

点击"处理 → 运行默认处理管线"自动执行三步标准流程：

1. 线性插值填充缺失值
2. 3σ 准则删除异常值
3. 移动平均（窗口=5）平滑

### 3.4 统计分析

#### 3.4.1 统计摘要

点击"分析 → 统计摘要"显示：

- 数据总体信息（行数、列数、缺失值数）
- 各传感器列的详细统计（均值、标准差、最大最小值、缺失数）

#### 3.4.2 相关性分析

点击"分析 → 相关性分析"计算传感器间的 Pearson 相关系数，并列出高相关对（|r| ≥ 0.7）。

#### 3.4.3 趋势分析

点击"分析 → 趋势分析"对每个传感器进行线性回归，判断趋势方向（上升/下降/平稳）并显示斜率、R² 和显著性 P 值。

### 3.5 可视化

#### 3.5.1 时间序列图

点击"视图 → 时间序列图"或工具栏按钮，绘制最多 6 个传感器的时间变化曲线，X 轴为时间戳（自动格式化），Y 轴为传感器数值，不同传感器用不同颜色区分。

[图 3-4 时间序列图]

#### 3.5.2 综合仪表盘

点击"视图 → 综合仪表盘"生成 2×2 布局的综合视图：

- 左上：时间序列折线图
- 右上：分布直方图
- 左下：箱线图
- 右下：相关性热力图

#### 3.5.3 批量导出图表

点击"视图 → 导出所有图表"一键生成所有类型图表并保存到 output 目录。

### 3.6 报告导出

**文本报告：**

点击"文件 → 导出分析报告"，选择 .txt 格式，生成包含数据概览、数值统计、相关性、异常检测和趋势分析的完整报告。

**Excel 报告：**

点击"文件 → 导出分析报告"，选择 .xlsx 格式，生成多 Sheet 的 Excel 报告：

- Sheet 1：报告总览（含分析结论摘要）
- Sheet 2：原始数据
- Sheet 3：统计摘要
- Sheet 4：相关性矩阵
- Sheet 5：异常检测详情

## 4. 命令行模式

除了图形界面，本系统还支持命令行批处理模式，适用于批量处理大量文件或集成到自动化流程中。

### 4.1 基本用法

```bash
python main.py --cli data.csv
```

### 4.2 批处理流程

命令行模式自动执行完整分析流程：

1. 导入 CSV 数据
2. 线性插值填充缺失值
3. 3σ 异常检测（输出检测报告）
4. 生成全部图表（保存到 output/ 目录）
5. 导出文本分析报告

### 4.3 参数说明

```bash
python main.py --help
```

输出：

```
usage: main.py [-h] [--cli CSV_FILE] [--gui] [--about] [--version]

物联网多源传感器数据分析系统

optional arguments:
  -h, --help          显示帮助信息
  --cli CSV_FILE      命令行批处理模式，指定 CSV 文件路径
  --gui               启动图形界面模式（默认）
  --about             显示版本信息
  --version           显示版本号
```

## 5. 数据源配置

### 5.1 CSV 文件格式

系统支持标准的 CSV 格式文件，约定如下：

- 第一行为列名
- 至少包含一列时间戳和至少一列传感器数值
- 时间戳列自动识别的列名：timestamp, time, datetime, date, 时间, 时间戳, 采集时间, 记录时间
- 数值列自动识别为传感器数据列

**示例：**

```
timestamp,temperature,humidity,pressure,light,wind_speed
2026-06-01 08:00:00,25.3,60.5,1013.2,450.1,2.5
2026-06-01 08:05:00,25.8,60.1,1013.1,480.3,3.1
```

### 5.2 串口配置

连接硬件传感器（如 Arduino、ESP32）时，需要配置以下参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 端口号 | 串口设备路径（如 COM3） | 无 |
| 波特率 | 通信速率 | 115200 |
| 数据位 | 数据帧位数 | 8 |
| 校验位 | 奇偶校验（N/E/O） | N |
| 停止位 | 停止位长度 | 1 |

### 5.3 MQTT 配置

连接物联网云平台时，需要配置以下参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| Broker 地址 | MQTT 服务器地址 | 无 |
| 端口 | MQTT 端口 | 1883 |
| 主题 | 订阅的传感器数据主题 | 无 |
| 客户端 ID | MQTT 客户端标识 | 自动生成 |

## 6. 常见问题

### 6.1 CSV 导入乱码

系统会依次尝试 utf-8、gbk、gb2312、utf-16、latin1 编码，通常能正确识别。如仍有问题，请将文件另存为 UTF-8 编码后重试。

### 6.2 图表中文字符显示为方框

系统已内置中文字体配置（SimHei、微软雅黑等），如您的系统不包含这些字体，请安装相应中文字体包。

### 6.3 异常值过多或过少

异常检测的灵敏度可通过代码调整：

- 3σ 准则：较严格，适合近似正态分布的数据
- IQR 方法：通过调整 multiplier 参数控制（默认为 1.5，越大越宽松）

### 6.4 大数据量处理建议

当数据量超过 10 万行时，建议：

1. 使用命令行模式而非图形界面
2. 使用重采样功能降低数据密度
3. 导出图表时选择需要的图表类型而非批量导出
